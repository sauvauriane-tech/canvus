Canvus URL Refactoring Test

This page tests the URL structure refactoring for Canvus.

URL Format Changes

Old Format

#file_abc123def456

Single file ID, no page information

New Format

#/file_abc123def456/page_1

Separate fileId and pageId with clear structure

Test 1: URL Parsing

Test 2: Page Switching URL Updates

Summary

The URL refactoring implementation includes:

  • ✓ New URL format: #/file_<fileId>/page_<pageId>
  • ✓ Backward compatibility with legacy format: #file_<fileId>
  • ✓ Automatic URL updates when switching pages
  • ✓ Page ID display in the UI
  • ✓ Copy page URL functionality
  • ✓ Proper initialization for both new and existing files

Manual Testing Instructions

  1. Open Canvus and create a new file
  2. Add multiple pages using the pages panel
  3. Switch between pages and verify the URL updates
  4. Click on the page info display to copy the page URL
  5. Paste the URL in a new tab and verify it loads the correct page
  6. Test with legacy URLs to ensure backward compatibility
